# Perguntas Frequentes - Iniciantes

Respostas diretas para dúvidas comuns.

## Básico

### O que são agentes?
Versões especializadas do Claude para tarefas específicas. Como ter um expert sempre disponível.

### Preciso saber programar?
Sim, mas não precisa ser expert. Os agentes ajudam você a melhorar.

### Qual agente começar?
`python-pro` se você programa em Python, `frontend-developer` se faz web.

### É grátis?
Depende do seu plano do Claude. Os agentes não custam extra.

## Uso Prático

### Por que meu código deu erro?
Provavelmente faltou contexto. Seja mais específico:
- Versão da linguagem
- Bibliotecas usadas
- Estrutura do projeto

### O agente não entendeu o que eu quero
Seja mais claro:
```
Ruim: "faça um site"
Bom: "faça um site de portfolio com React, 3 seções, responsivo"
```

### Posso confiar no código gerado?
Sempre revise e teste. Os agentes são bons mas não perfeitos.

### Como combinar agentes?
Use em sequência:
1. Arquiteto projeta
2. Desenvolvedor implementa
3. Tester valida
4. Security audita

## Problemas Comuns

### "Command not found"
Você precisa ter Node.js instalado para usar os comandos npm.

### Código muito complexo
Peça simplicidade: "faça a versão mais simples possível"

### Não sei qual agente usar
Comece com o `backend-architect`, ele indica os próximos passos.

### O agente está repetindo erros
Dê feedback claro: "o erro X persiste, tente outra abordagem"

## Dicas

### Como aprender mais rápido?
1. Faça projetos pequenos
2. Compare com seu código
3. Peça explicações
4. Refaça sozinho depois

### Vale a pena para iniciantes?
Sim! É como ter um mentor 24/7.

### Posso criar meus agentes?
Sim, mas primeiro domine os existentes.

### Qual workflow usar?
Sempre: Research → Planning → Implementation → Testing

## Comandos Úteis

### Para pensar mais
```
"think hard sobre [problema]"
```

### Para TDD
```
"escreva testes primeiro, NÃO implemente"
```

### Para explicações
```
"explique cada decisão técnica"
```

### Para simplicidade
```
"faça a versão mais simples que funciona"
```

## Erros de Iniciante

### Pular planning
Sempre planeje antes de codificar.

### Usar agente errado
Backend para backend, frontend para frontend.

### Não testar
Sempre valide o código gerado.

### Aceitar tudo
Questione, entenda, aprenda.
