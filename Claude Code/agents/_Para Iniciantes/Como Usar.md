# Como Usar Agentes - Guia Iniciante

Passo a passo simples para começar.

## 1. Escolha o Agente Certo

| O que você quer fazer? | Use este agente |
|------------------------|-----------------|
| Criar uma API | `backend-architect` |
| Escrever Python | `python-pro` |
| Fazer interface React | `frontend-developer` |
| Achar bugs de segurança | `security-auditor` |
| Configurar deploy | `deployment-engineer` |

## 2. Use o Workflow Certo

A Anthropic descobriu que esta ordem funciona melhor:

```
1. Research (pesquise)
2. Planning (planeje)
3. Implementation (implemente)
4. Testing (teste)
```

**Importante:** Não pule direto para o código!

## 3. Exemplo Prático

Vamos criar uma API de tarefas:

### Passo 1 - Pesquise
```
"research o projeto e veja se já existe algo similar"
```

### Passo 2 - Planeje
```
"think hard sobre a melhor arquitetura para uma API de tarefas"
```

### Passo 3 - Implemente
```
"implemente a API usando FastAPI e PostgreSQL"
```

### Passo 4 - Teste
```
"escreva testes para garantir que tudo funciona"
```

## 4. Comandos "Think"

Quando precisar de análise mais profunda:

- `think` - análise básica
- `think hard` - análise profunda
- `think harder` - considera alternativas
- `ultrathink` - máximo de análise

Use para decisões importantes!

## 5. Arquivo CLAUDE.md

Crie na raiz do projeto:

```markdown
# CLAUDE.md

Projeto: API de tarefas
Stack: FastAPI + PostgreSQL
Python: 3.11
Convenções: snake_case, type hints
```

O Claude vai ler isso sempre.

## 6. TDD (Test-Driven Development)

O jeito favorito da Anthropic:

```bash
# 1. Teste primeiro
"escreva testes para criar usuário. NÃO implemente ainda"

# 2. Código mínimo
"agora implemente só o necessário para passar os testes"

# 3. Melhore
"refatore mantendo os testes passando"
```

## Dicas para Iniciantes

### Comece Pequeno
- Uma função simples
- Um endpoint básico
- Um componente React

### Seja Específico
```
Ruim: "crie uma API"
Bom: "crie uma API REST de produtos com CRUD usando FastAPI"
```

### Compare e Aprenda
Veja o que o agente fez diferente do que você faria. Essa é a melhor forma de aprender.

### Não Tenha Medo de Errar
Os agentes são ferramentas. Teste, quebre, aprenda.

## Próximo Passo

Depois de entender o básico:
1. Teste diferentes agentes
2. Combine múltiplos agentes
3. Crie seus próprios agentes

Lembre-se: é uma ferramenta para te fazer melhor, não para te substituir.
